# Frameworkization Roadmap — Affiliate Content Pipeline

**Created**: 2026-07-03
**Status**: Strategic target, not immediate scaffold
**Source**: User direction + `scaffold-framework` AIWG guidance + intake artifacts

## Strategic Intent

The local affiliate/content pipeline may eventually become its own AIWG framework inside the user's AIWG fork. That framework would package the proven operating system for affiliate/product-content creation:

- rules
- skills
- workflows/flows
- runtime behaviors
- specialist agents
- daemon-like agents that automate the pipeline
- templates and playbooks

The framework should not be scaffolded prematurely. The local MVP must first prove that it can produce real return on Higgsfield credits, LLM/token spend, data-source subscriptions, and operator time.

## Economic Gate

The frameworkization gate is economic, not aesthetic.

Do **not** graduate into a framework until the system demonstrates:

1. Positive contribution margin after Higgsfield credits, token spend, data-source subscriptions, and operator time.
2. Repeatable product/niche selection that produces winners, not one-off luck.
3. A validated benchmark loop that improves creative quality over time.
4. A stable memory loop that captures lessons without context rot.
5. Compliance gates that prevent risky or platform-violating content.

### Working Metrics

| Metric | Why it matters |
|---|---|
| Higgsfield credits/video | Primary generation cost. |
| Token spend/video | Script/research/review cost. |
| Tool spend/niche | Market-intelligence and data-source cost. |
| Operator minutes/video | Solo-operator throughput constraint. |
| Cost per generated video | Raw production efficiency. |
| Cost per publishable video | Quality + compliance efficiency. |
| Cost per winning video | True creative efficiency. |
| Affiliate clicks/conversions | Market response. |
| Gross commission revenue | Top-line monetization. |
| Contribution margin | Whether the loop is worth scaling. |
| MMR/MRR | Monetized monthly revenue / monthly recurring revenue shorthand until formal accounting is defined. |

## AIWG Framework Shape

Candidate framework name: `affiliate-content-ops` or `creator-commerce-ops`.

Candidate phases:

1. **Discovery** — market/product/niche research.
2. **Strategy** — niche selection, persona strategy, offer selection, benchmark set.
3. **Production** — scripts, personas, Higgsfield creative generation.
4. **Validation** — Virality Predictor, benchmark comparison, compliance gate.
5. **Distribution** — publish/test, platform-specific workflows.
6. **Analytics** — performance capture, attribution, ROI and MMR/MRR.
7. **Optimization** — memory consolidation, scale/kill/pivot decisions.

## Candidate Rules

- `research-before-publish`: no product/niche/content batch without source-backed research.
- `benchmark-before-scale`: compare against top creator/product gold sets before scaling.
- `compliance-before-publish`: every creative must pass affiliate disclosure + claim safety.
- `no-medical-claims`: health/beauty content cannot imply diagnosis/treatment/weight-loss claims.
- `ai-persona-honesty`: no impersonation or false firsthand claims.
- `roi-before-frameworkization`: no framework graduation until positive contribution margin and repeatability.
- `memory-without-rot`: capture facts/events/procedures with dedup, temporal validity, and relevance-gated retrieval.

## Candidate Agents

- **Market Researcher** — identifies niches/products/trends.
- **Product Scout** — scores affiliate offers and product-market fit.
- **Benchmark Analyst** — maintains the gold-set of top videos/products.
- **Persona Director** — manages AI persona consistency and ethics.
- **Scriptwriter** — writes hooks/scripts against proven templates.
- **Creative Producer** — drives Higgsfield generation.
- **Compliance Reviewer** — reviews FTC/platform/claim safety.
- **Performance Analyst** — reads metrics and recommends iterations.
- **Memory Curator** — compresses/consolidates lessons and prevents rot.
- **Pipeline Orchestrator** — coordinates daily loop.

## Candidate Behaviors / Daemons

- **Trend Watcher**: monitors product/niche signals daily.
- **Gold-Set Refresher**: refreshes benchmark examples and detects shifts.
- **Daily Batch Producer**: prepares the 20-video generation queue.
- **Compliance Gatekeeper**: blocks publish when disclosure/claim requirements fail.
- **Performance Harvester**: pulls post-publish results and stores them.
- **Memory Consolidator**: dedups and promotes episodic results into semantic/procedural lessons.
- **ROI Sentinel**: stops or warns on negative return from credits/tokens/tools.

## Promotion Path

1. **Local MVP**: prove one niche can run end-to-end locally.
2. **Project-local bundle**: pilot reusable rules/skills/agents under `.aiwg/` once patterns stabilize.
3. **Framework candidate**: use `aiwg scaffold-framework` only after repeated success.
4. **AIWG fork promotion**: graduate into `agentic/code/frameworks/<name>/` after the package has real lifecycle phases and reusable artifacts.

## Current Recommendation

Stay in **Local MVP** until the first niche has:

- ≥1 full production week of data,
- tracked generation costs,
- tracked affiliate outcomes,
- at least one validated creative pattern,
- clear memory-capture rules,
- no unresolved compliance blockers.

Then revisit project-local bundle scaffolding before full framework scaffolding.
